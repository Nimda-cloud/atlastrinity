# План виправлення: Перемикання на Windsurf через збій Copilot API

## Опис проблеми

Логи (`brain.log`) підтверджують, що GitHub Copilot API повертає `500 Internal Server Error`. Це призводить до краху сегментатора запитів та неможливості виконання завдань Atlas. Перший запит пройшов, бо тоді використовувався провайдер `windsurf`.

## План дій

### Етап 1: Модифікація конфігурації

1. Оновити файл `config/config.yaml.template`, встановивши `provider: "windsurf"`.
2. Оновити моделі за замовчуванням на `deepseek-v3` для стабільності.

### Етап 2: Синхронізація

1. Запустити скрипт синхронізації, щоб оновити активний конфіг у `~/.config/atlastrinity/config.yaml`.
   `python3 scripts/setup_dev.py --sync-configs`

### Етап 3: Верифікація

1. Перевірити лог ініціалізації (Atlas має написати `Initialized with models: windsurf:deepseek-v3`).
2. Виконати тестовий запит для перевірки сегментації.

## Очікуваний результат

Система відновить роботу, використовуючи API Windsurf (Codeium), який на даний момент працює без помилок.
